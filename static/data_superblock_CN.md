
## 超级块

> 如无必要，请避免创建超级块。不规范的超级块格式可能导致笔记异常！

思源笔记（Siyuan）使用的 **kramdown 超级块（Super Block）** 是一种极具特色的扩展语法，它通过将多个块（Block）包裹在特定的容器中，实现了 Markdown 原生语法难以做到的 **多栏布局（Column）** 和 **复杂嵌套排版**。

## 1. 基础语法结构
超级块的本质是一对 **三个大括号** 组成的容器，其基本结构如下：

```markdown
{{{类型
内容块 A
{: 内容块A的属性扩展}

内容块 B
{: 内容块B的属性扩展}

}}}
{: 属性扩展}
```

* **`{{{` 与 `}}}`**：定义了超级块的边界。
* **类型标识**：
    * `col`：表示**横向排布**（Columns），其内部的直接子块会并列成多列。
    * `row`：表示**纵向排布**（Rows），其内部的直接子块会垂直堆叠。
* **属性扩展 `{: ...}`**：紧跟在 `}}}` 之后，用于存储该超级块的唯一 ID（如 `id="20260317114025-hrr7cvu"`）和更新时间等元数据。

---

## 2. 核心排版逻辑

### A. 多列布局 (`{{{col ... }}}`)
当使用 `col` 时，超级块会将内部的**每一个独立块**视为一列。
* 如果你放入两个段落块，它们会左右并排。
* 如果你放入一个列表块和一个段落块，它们依然会左右并排。

### B. 嵌套组合（复杂结构的关键）
超级块最强大的地方在于可以**相互嵌套**。
1.  外层是一个 `{{{col ... }}}`（分为左右两部分）。
2.  左侧是一个普通的 **无序列表块**。
3.  右侧嵌套了一个 `{{{row ... }}}`。
    * 在这个 `row` 内部，又包含了 **段落块** 和 **引述块**，使它们在右侧这一列中上下排列。

---

## 3. 属性挂载规则
在思源的 kramdown 格式中，**属性（Attributes）** 的位置至关重要：

1.  **块内属性**：紧跟在内容块下方，定义该具体块（如段落、列表项）的 ID。
2.  **超级块属性**：紧跟在结尾的 `}}}` 之后，定义整个容器的 ID。
3.  **空行要求**：通常每个块与属性之间、以及块与块之间会有换行，以确保解析器能准确识别块的分界。

---

## 4. 实例对比总结

| 布局类型 | 表现形式 | 适用场景 |
| :--- | :--- | :--- |
| **单层 col** | `A | B` | 简单的左右对比、分栏笔记。 |
| **单层 row** | `A / B` | 逻辑归组，虽然视觉上类似普通排列，但在数据库中被视为一个整体。 |
| **嵌套 col + row** | `A | [B/C]` | 复杂的看板布局、侧边栏说明、图文混排。 |

---

## 提示与建议
* **ID 的重要性**：思源笔记依赖 `id` 进行块级引用。如果你在手动编写此类格式，请务必保留 `id` 属性，否则在笔记中无法被正确定位或引用。
* **手动编辑风险**：虽然这种格式可以手动编写，但稍有不慎（如少了一个大括号或空行不对）会导致渲染失败。建议优先在思源编辑器内通过 **拖拽块边框** 的方式生成，然后进入“块源码编辑”模式查看学习。
